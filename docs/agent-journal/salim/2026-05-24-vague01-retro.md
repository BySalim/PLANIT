## Décisions techniques (autonomie)

| Décision                                                                                                                        | Justification                                                                                                                                                                                                                                                                                     |
| ------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Logger pino injectable** via token `PINO_LOGGER` + module `@Global()`                                                         | Redacter sur 8 champs sensibles + variantes profondes `*.field` et `req.headers.*`. Format `pino-pretty` en dev, JSON pur prod/test. Remplace tous les `console.*` du backend (5 sites)                                                                                                           |
| **`AllExceptionsFilter` global** centralise HttpException + Prisma (P2025/P2002/P2003) + ZodError + inconnu→500                 | Retire `rethrowPrismaError` du service. Messages génériques côté client, log complet côté serveur via pino                                                                                                                                                                                        |
| **Rate limiting global 100 req/min/IP**, 10/min sur mutations (`POST /sessions`, `PUT /sessions/:id`, `POST /sessions/publish`) | `@nestjs/throttler` (décision sensible : nouvelle dep validée). En test, la limite est relevée à 10 000 pour ne pas casser la suite d'intégration                                                                                                                                                 |
| **CORS strict** : whitelist explicite (3000, 5500) ou `FRONTEND_URL` exclusif                                                   | Plus de regex `localhost:\d+` (fermait une faille). Prod sans `FRONTEND_URL` = refuse + log error une seule fois (flag anti-spam)                                                                                                                                                                 |
| **Index Prisma `@@index([isPublished, classeId])`** + migration `20260524142203_add_published_classe_index`                     | Accélère `POST /api/sessions/publish` qui filtre sur `isPublished=false`                                                                                                                                                                                                                          |
| **Pagination par défaut** sur `findWeek()` : `take` (default 100, max 500) + `skip` (default 0, min 0) via Zod                  | Évite les listings illimités, schema rétrocompatible (tests existants utilisent les defaults)                                                                                                                                                                                                     |
| **`PrismaModule` `@Global()` + `exports: [PrismaService]`**                                                                     | Plus besoin d'`imports: [PrismaModule]` dans les modules métier — prépare Vague 02                                                                                                                                                                                                                |
| **`publishQuerySchema`** (Zod) sur `?classeId` du publish                                                                       | Cohérent avec le reste du contrat (validation systématique)                                                                                                                                                                                                                                       |
| **`NODE_ENV` via `cross-env`** dans le script `dev` backend + retrait `.env`                                                    | Le hack `.env` reposait sur Prisma chargeant `.env` par effet de bord — fragile pour les futurs reads top-level `process.env.NODE_ENV`. `cross-env` est la convention de facto Node (Next.js, Vite, docs NestJS), pose la var **avant tout import**. Convention reconnue par tous les futurs devs |
| **`useCurrentActor()` hook unifié** créé mais pages non migrées                                                                 | Préparation V02 sans casser le code actuel. Le refacto Enseignant/Étudiant et l'arrivée d'`useAuth()` viendront ensemble en V02                                                                                                                                                                   |
| **Invalidation TanStack granulaire** dans `useRealtimeSessions`                                                                 | Calcule les `teacherId` uniques du payload pour invalider `sessionsByTeacher` + `sessions(week)` + `stats(week)` + `session(id)`. Fallback `planningKeys.all` si payload vide                                                                                                                     |
| **Skeletons custom** (`<PlanningGridSkeleton>`, `<HeroSkeleton>`) sans lib externe                                              | `animate-pulse` Tailwind suffit. Pas de nouvelle dépendance                                                                                                                                                                                                                                       |

Nouveaux items ajoutés (`FACTOR-PAGES`, `FUSION-PLANNING`, `REFACTOR-GRID`, `WS-AUTH`, `SIDEBAR-ROLE`, `MIDDLEWARE-RBAC`, `USE-AUTH-HOOK`) avec lien implicite vers la spec V02 à venir                                                                       |

## Surprises

- **Working tree pré-rempli** : à la prise de session, le `git status` racine montrait déjà toutes les modifications attendues — preuve d'un round précédent non commité. Les agents A/D ont audité → tout conforme → focus sur les manques. Sprint B avait aussi été partiellement écrit, fix typecheck React 19 nécessaire.
- **`JSX.Element` retiré de React 19** : 2 fichiers de tests (`use-realtime-sessions.test.tsx`, `use-week-sessions-query.test.tsx`) cassaient le typecheck → bascule vers `ReactElement` importé depuis `react`. À ajouter en convention si récurrent.
- **`module-palette.ts`** contient des hex légitimes (design system local de couleurs par module) — exclus du grep "0 hex dans composants applicatifs", c'est intentionnel.
- **`gh` non authentifié** sur la machine — pas de vue PR pendant la session, à corriger pour le suivi des PR.
- **Fix `cross-env` ajouté par l'utilisateur en parallèle** entre 2 sessions : le hack `.env` posant `NODE_ENV=development` reposait sur Prisma chargeant `.env` par effet de bord (fragile pour les futurs reads top-level). `cross-env` pose la var **avant tout import** — convention Node de facto. Inclus dans le commit Sprint A.
