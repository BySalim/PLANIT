## Décisions techniques

| Décision                                            | Justification                                                                                                                                                                                       |
| --------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Design tokens 0.3 = no-op                           | `packages/design-tokens/src/colors.ts` contenait déjà les 30 couleurs canoniques depuis le bootstrap. La tâche était complète.                                                                      |
| `packages/config/typescript/base.json` auto-contenu | Le `"extends": "../../../tsconfig.base.json"` original cassait ts-node via les symlinks pnpm (TS5083). Solution : inliner tous les compilerOptions, plus fragile extension relative.                |
| Seed sans `@planit/utils`                           | ts-node ne peut pas compiler les packages TypeScript du workspace via les symlinks pnpm sans configuration spéciale. Les dates sont calculées en UTC direct (Africa/Dakar = UTC+0).                 |
| `useId()` dans PlanitLogo                           | Les IDs de gradient SVG doivent être uniques par instance. `useId()` est hydration-safe et fonctionne en RSC/SSR.                                                                                   |
| `process.cwd()` dans vitest.config.ts               | Évite l'ambiguïté ESM/CJS de `__dirname` dans les fichiers de config vitest.                                                                                                                        |
| Playwright e2e non bloquant                         | `playwright install chromium` a échoué (problème réseau). La CI ne requiert pas `test:e2e` — seul `pnpm test` (vitest) est bloquant. E2E sera installé localement lors du premier smoke navigateur. |

## Surprises

- **pnpm symlink + ts-node** : L'extends relatif `../../../` dans `base.json` se résout depuis le répertoire du symlink et non le répertoire réel. Problème subtil, long à diagnostiquer. Fix définitif = fichier auto-contenu.
- **Turbo scope scope** : Turbo détecte le package courant selon le CWD shell. Un `cd apps/backend` résiduel faisait tourner `typecheck` uniquement sur le backend. À documenter pour l'équipe.
- **`prisma.seed` dans package.json** : Déprécié en Prisma 7 (warning), toujours fonctionnel en v6. Migration vers `prisma.config.ts` à planifier en tech-debt.
- **Design tokens déjà complets** : La tâche 0.3 était déjà faite depuis le bootstrap. Le plan de Vague 01 ne le savait pas.
