## 2. Décisions techniques (prises en autonomie)

- **Stratégie 2 endpoints parallèles** : `/api/sessions` V01 reste **intact**
  (consommé par le web V01 jusqu'au LOT 3), `/api/v2/sessions` V02 est
  nouveau (smart dirty + multi-classes). Pas de double-write — le seed
  alimente déjà les deux modèles. Les séances créées via V02 ne sont pas
  visibles par le web V01 (acceptable : LOT 3 bascule rapidement).
- **TD-028 fixé** : ajout `main`/`types`/`exports map ./json-stable` dans
  `packages/utils/package.json`. Résolution `moduleResolution: "node"` du
  tsconfig nest trouve maintenant `@planit/utils` (sans toucher au config
  partagé `@planit/config/typescript/nest`). Solution la moins invasive.
- **Migration mineure** `20260525130000_v02_relax_seance_nullables` :
  `salleId` et `teacherId` passent nullable sur `Seance`. Permet à un
  EVENEMENT V02 de ne pas avoir d'enseignant interne (intervenant externe)
  ni de salle. V01 service mis à jour pour gérer `teacher | null` dans
  `concernedUserIds`. Mapper V01 lance une erreur explicite si rencontré
  (ne devrait jamais arriver via endpoint V01).
- **Contracts `createEnseignantSchema` adapté** : prend `email + password
  - nomComplet + …`au lieu d'un`userId` pré-existant. Le service crée
    atomiquement le User (rôle ENSEIGNANT + argon2id) et l'Enseignant (1-1)
    dans une transaction Prisma. Cohérent avec le scénario démo (RP crée
    des enseignants depuis l'UI sans passer par un flow register séparé).
- **Smart dirty (ADR-0008)** dans `seance-v2/dirty-detection.ts` :
  `toComparable`, `computeSnapshot` (= `stableStringify(extractComparable)`),
  `isDirty`. Au PUT, on recalcule `hasUnpublishedChanges` en comparant
  l'état post-update au snapshot stocké. Au PUBLISH, on écrit le snapshot
  et passe à `hasUnpublishedChanges=false`. Le test smart dirty (étape 14
  du scénario démo) passe vert.
- **V01 mirror systématique** dans `SeanceV2Service.create` : pour chaque
  séance V02 créée, on écrit aussi les colonnes V01 (`type` mappé via
  `mapV2ToV01Type`, `classeId = classeIds[0]`, `teacherId = enseignant.userId`
  ou `createdByUserId` placeholder pour EVENEMENT). Permet au seed et au
  web V01 de continuer à fonctionner en parallèle.
- **Filtrage V02 strict** : tous les endpoints V02 ajoutent
  `where: { typeV2: { not: null } }` pour exclure les rows legacy V01 sans
  backfill. Si une séance V01 n'a pas été migrée en V02 (`typeV2=null`),
  elle est invisible côté V02 — pas d'erreur, juste absente.
- **WS multi-classes (B.11)** : recalcul des destinataires via 2 queries
  Prisma parallèles (enseignants liés par `enseignantId → User`,
  étudiants des classes via `SeanceClasse → classeId`). Dédup via Set.
  Réutilise `WsGateway.emitSessionPublished` qui accepte désormais
  `(SessionDto | SessionV2Dto)[]`.

## 3. Décisions soumises à validation Salim

(auto-validation tech-lead pour cette session — traçabilité revue PR)

- **2 endpoints parallèles** vs migration destructive — choix tactique

  un invariant V01 (NOT NULL) mais sans impact pratique (le V01 Zod

  invité, on séparera.
- **`mapV2ToV01Type`** : algorithme déterministe pour rester cohérent
  avec le seed. Documenté inline dans `seance-v2.mapper.ts`.

## 6. Surprises

- **Prisma `dll` lock Windows** : `prisma generate` a échoué deux fois
  (EPERM rename query_engine-windows.dll.node) à cause de processus node
  orphelins (ts-node tests précédents) qui tenaient le binaire. Solution :
  réessayer plus tard sans tuer les processus user (refus du classifier).
- **`exactOptionalPropertyTypes: true`** : forcé à utiliser des spreads
  conditionnels `...(value !== undefined ? { value } : {})` dans plusieurs
  fichiers (enseignants controller, seance-v2 service). Verbeux mais
  protège contre les leakage de `undefined`.
- **`Prisma.SeanceUpdateInput` vs `SeanceUncheckedUpdateInput`** : la
  première expose `salle: { connect: { id } }` (relations), la seconde
  expose `salleId` (FK scalaire). Switch sur `Unchecked*` côté V02 service
  pour pouvoir écrire `salleId` directement — plus simple.
- **`UE` Prisma client = `prisma.uE`** (camelCase auto). Pas de conflit avec
  `@nestjs/common.Module` puisque je nomme la classe `ModulesService` (pluriel).
- **Tests `dirty-detection` ont confirmé le pattern ADR-0008** : permuter
  les `classeIds` ne change pas le snapshot (tri lexicographique en
  amont). Modifier 1 minute → dirty. Snapshot null = toujours dirty.
