## Décisions techniques

- **Découpage en 4 PR locales** sur `feat/libasse` (commits successifs) plutôt
  qu'une PR monolithique : foundations + display → primitives → mutations →
  polish/check/journal. Chaque chunk testé en navigateur via le MCP
  `Claude_Preview` avant commit. PR finale `feat/libasse → develop` à ouvrir
  manuellement après push.
- **`@tanstack/react-query` v5** + clé partagée `planningKeys.all` : toute
  mutation (create/update/publish) invalide à la fois la liste des séances
  ET les stats de la semaine. Pas de cache manuel.
- **`react-hook-form` + `zodResolver`** avec un **schema de formulaire dédié**
  (`sessionFormSchema`) plutôt que `createSessionSchema` directement, parce
  que le form expose `date + startTime + endTime` recombinés en ISO UTC
  côté `onSubmit`. Première itération avec `zodResolver(createSessionSchema)`
  bloquait silencieusement la soumission — corrigé pendant PR3.
- **Africa/Dakar = UTC+0** : on traite les valeurs de formulaire comme déjà
  UTC (`${date}T${time}:00.000Z`) — cohérent avec la seed backend et évite
  un fuseau de DST.
- **UI primitives dans `apps/web/src/components/ui/`** (Button/Modal/Drawer/
  Input/Select/FormField) — pas dans `@planit/ui` pour rester rapide en V1
  (web-only). Migration possible vers `@planit/ui` quand Mobile en aura
  besoin (Vague 04).
- **Listes hardcodées** (`apps/web/src/lib/seed-refs.ts`) pour classes /
  modules / salles / enseignants — pas d'endpoint REST pour ces référentiels
  encore. Tracé en TD-010.
- **Drag & drop reporté V2** (TD-009) — le proto Design embarque un système
  complexe (snap, resize, undo/redo) hors périmètre V1.
- **`packages/contracts` re-exporte `z` (zod)** — évite d'ajouter `zod` comme
  dépendance directe d'`apps/web` alors qu'elle est déjà transitive. Permet
  d'utiliser le même runtime zod côté front pour les schemas de formulaire.

## Décisions soumises à validation

- Stack TanStack Query / react-hook-form / @hookform/resolvers / date-fns —
  4 dépendances web ajoutées (chunk dépendances isolé dans PR1, commit

- Re-export `z` depuis `@planit/contracts` — modif d'un package partagé.

  futures sessions).
- `apps/web/src/app/(planit)/layout.tsx` wrappé d'un `<QueryProvider>` —
  impact LOTs 3 et 4 qui réutiliseront le même layout (bénéfique).

## Surprises

- **Port 3000 squatté** par un node orphelin au démarrage — résolu en killant
  via PowerShell. Plus tard découverte de 27 processus PLANIT-related restés
  ouverts sur la machine, nettoyés en bloc pour libérer le query engine
  Prisma (rebuild bloqué par `EPERM` sur `query_engine-windows.dll.node`).
- **`zodResolver(createSessionSchema)` rejette silencieusement** quand la
  forme du form ne matche pas le schema. Détecté au premier smoke test
  PR3, corrigé en introduisant `sessionFormSchema` (form-only) puis
  recombinant en `CreateSessionDto` à l'envoi.
- **Erreurs Next.js HMR persistantes** : le buffer console accumule les
  erreurs de compilation historiques sans dédoublonnage — 180 entrées
  identiques affichées pour des modules qui résolvent maintenant
  correctement. Confusion résolue en vérifiant le code source réel + les
  smoke tests fonctionnels.
- **Couleurs design** : Salim a aligné `--color-primary` / `--color-accent`
  sur les valeurs Illustrator exactes (`#593114` / `#EE7023`) — la grille
  utilise donc ces tokens sans hex en dur.
