## 2. Décisions techniques

### Approche responsive MobileShell (D.2)

Décision : responsive switch à `md` (768px) via classes Tailwind v4 `max-md:*`.

- **Mobile (< 768px)** : comportement inchangé — `max-w-md mx-auto`, tab bar flottante, header compact.
- **Desktop (≥ 768px)** : `DesktopSidebar` (w-64), suppression `max-w-md`, tab bar + header masqués via `md:hidden`.

La `DesktopSidebar` contient : logo PLANIT, liens Accueil/Planning avec `aria-current`, badge notification, user info (initiales + nom + rôle) + bouton déconnexion avec `router.push('/login')`.

Tailwind v4 supporte les variantes `max-md:` nativement — validé par typecheck + lint zéro erreur.

### Découplage du logout (D.2 / LOT 6 cohérence)

Ajout de `useAuth().logout` + `useRouter()` dans les MobileShells pour que la DesktopSidebar puisse déclencher `POST /auth/logout` puis redirect `/login`. Le pattern est identique à `dev-auth-badge.tsx`.

### Fix date locale (TD-V02-LOT3-B)

`session-detail-drawer.tsx` ligne 523 manquait `{ locale: fr }` sur `format()`. Ajout de l'import `fr` de `date-fns/locale` et passage de la locale — la date s'affiche désormais en français.

### D.5 — WS cross-test

La `PlanningUpdateModal` déjà en place dans les pages home et planning des deux acteurs couvre D.5 — elle s'ouvre lorsqu'un event `session:published` arrive via `useRealtimeSessions`. Pas de code supplémentaire nécessaire.
